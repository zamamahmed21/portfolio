Full Screen Browser Launcher

Problem Statement :

The project aims to develop a user-friendly desktop application that simplifies the process of opening multiple URLs on different external screens with options like kiosk mode, full screen, or standard mode, and auto-launching at startup based on user settings.

Technologies Used

  • Python
  • PySide2
  • screeninfo   (Python library for getting the details of multiple connected screens.)
  • CTypes   (Python library for calling C functions and interacting with Windows APIs.)
  • PyAutoGUI   (Python library for controlling the mouse and keyboard programmatically.)
  • Win32 Libraries   (Windows-specific libraries for GUI manipulation and window management.)
  • Pathlib   (Python module for object-oriented file path manipulation.)
  • JSON   (Used for saving and loading application settings.)
  • Windows Registry (WinReg)   (Used for modifying Windows registry settings.)
  • Subprocess   (Python module for running system commands and processes.)

Key Features:

  • Users can input URLs and specify the screen where each URL should be opened.
  • Option to choose how the browser window will open (kiosk mode, full screen, or standard mode).
  • The application supports auto-launch at startup if the user checks the 'auto-launch' option.
  • Two additional URLs can be opened in the background without interfering with the primary windows.

Challenges:

Ensuring the browser opened in kiosk mode on external screens was initially problematic. The application could open URLs on the main screen, but external screens posed difficulties in both positioning and enabling kiosk mode.

Solution:

To overcome the challenge of opening URLs on external screens in kiosk mode, I utilized the 'screeninfo' library to detect connected screens and calculate their positions. Using Selenium, I opened the browser and used the window position function to move the browser to the appropriate screen. For kiosk mode, specific command-line arguments (e.g., '--kiosk' for Chromium-based browsers) were passed during browser launch. This allowed the browser to open correctly on external screens and in the required display mode.

Outcome:

The app successfully enables users to open multiple URLs on different screens, select display modes, and auto-launch the app on startup, improving both convenience and productivity.
