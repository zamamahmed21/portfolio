Recording App

Problem Statement :

The project aims to develop a user-friendly desktop application that records both system and microphone audio simultaneously on macOS, while allowing users to run custom scripts on the recorded audio.

Technologies Used

  • Python
  • PySide2
  • BlackHole   (for audio rerouting on macOS)
  • Audio MIDI Setup   (for configuring internal and external audio sources)
  • FFmpeg   (for saving audio in different formats)

Key Features:

  • Record both internal system audio and external microphone audio simultaneously on macOS.
  • Execute user-provided scripts on the recorded audio.
  • Save audio in WAV, M4P, and MP3 formats.
  • Automatically configure BlackHole and Audio MIDI Setup for seamless audio recording.
  • Undo all audio routing configurations when the app is closed.

Challenges:

In macOS, recording system audio is more complex compared to Windows due to security restrictions. Granting permissions and setting up internal audio recording required additional configuration through external apps.

Solution:

To overcome this challenge, I developed an automated script that installs BlackHole (an audio rerouting tool), configures a multi-output device in Audio MIDI Setup, and grants necessary permissions for internal and external audio recording. This script runs during the app setup, ensuring the user doesnâ€™t need to perform any manual configurations. The script also reverts all settings when the app is closed.

Outcome:

Users were able to run the application without any additional effort, successfully record both internal and external audio, and save the output in multiple formats (WAV, M4P, MP3). The app also allows users to run their own scripts on the audio and displays the results of those scripts within the application.
